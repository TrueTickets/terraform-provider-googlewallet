# Google Wallet API Interface Documentation

## Overview

The Google Wallet API provides endpoints for managing Issuers and Permissions. This document captures the API contract for the terraform-provider-googlewallet implementation.

**Base URL**: `https://walletobjects.googleapis.com/walletobjects/v1/`

**OAuth Scope**: `https://www.googleapis.com/auth/wallet_object.issuer`

---

## Issuer Endpoints

### Create Issuer (insert)

**POST** `/walletobjects/v1/issuer`

Creates a new Issuer. The `issuerId` is auto-generated by Google.

**Request Body:**
```json
{
  "name": "string (required)",
  "contactInfo": {
    "name": "string (optional)",
    "email": "string (optional)",
    "phone": "string (optional)",
    "alertsEmails": ["string (optional)"]
  },
  "homepageUrl": "string (optional)",
  "callbackOptions": {
    "url": "string (HTTPS, publicly accessible)"
  },
  "smartTapMerchantData": {
    "smartTapMerchantId": "string",
    "authenticationKeys": [...]
  }
}
```

**Response:**
```json
{
  "issuerId": "int64 (string format)",
  "name": "string",
  "contactInfo": {...},
  "homepageUrl": "string",
  "callbackOptions": {...},
  "smartTapMerchantData": {...}
}
```

**Notes:**
- `issuerId` is assigned by Google and returned in response
- `smartTapMerchantData` requires Google partnership program

---

### Get Issuer

**GET** `/walletobjects/v1/issuer/{issuerId}`

Retrieves a specific Issuer by ID.

**Response:** Same as Create response

---

### List Issuers

**GET** `/walletobjects/v1/issuer`

Lists all Issuers accessible by the authenticated credentials.

**Response:**
```json
{
  "resources": [
    {
      "issuerId": "string",
      "name": "string",
      "contactInfo": {...},
      "homepageUrl": "string"
    }
  ]
}
```

---

### Update Issuer (patch)

**PATCH** `/walletobjects/v1/issuer/{resourceId}`

Partially updates an Issuer. Only specified fields are modified.

**Request Body:** Any subset of Issuer fields

**Response:** Updated Issuer object

---

### Update Issuer (full replacement)

**PUT** `/walletobjects/v1/issuer/{resourceId}`

Fully replaces an Issuer. All fields not specified are reset to defaults.

**Warning:** Use PATCH for partial updates to avoid data loss.

---

### Delete Issuer

**NOT SUPPORTED**

The Google Wallet API does not support deleting Issuers. The Terraform provider will implement "soft delete" by removing from state and logging a warning.

## Permissions Endpoints

### Get Permissions

**GET** `/walletobjects/v1/permissions/{issuerId}`

Retrieves current permissions for an Issuer.

**Response:**
```json
{
  "issuerId": "int64 (string format)",
  "permissions": [
    {
      "emailAddress": "string",
      "role": "OWNER | READER | WRITER"
    }
  ]
}
```

---

### Update Permissions

**PUT** `/walletobjects/v1/permissions/{resourceId}`

Updates (replaces) all permissions for an Issuer.

**CRITICAL:** This operation is authoritative - it replaces ALL existing permissions with the provided list. Always fetch existing permissions first, modify, and send the complete list.

**Request Body:**
```json
{
  "issuerId": "string",
  "permissions": [
    {
      "emailAddress": "string",
      "role": "OWNER | READER | WRITER"
    }
  ]
}
```

**Response:** Same as request body

**Valid Roles:**
| Role | Description |
|------|-------------|
| `OWNER` | Full administrative access |
| `READER` | View-only access |
| `WRITER` | Modification permissions |

**Note:** Lowercase variants (owner, reader, writer) are deprecated - use uppercase.

## Authentication

### Service Account Setup

1. Enable Google Wallet API in Google Cloud Console
2. Create service account and download JSON key
3. Add service account to Issuer in Google Pay & Wallet Console:
   - Go to Console → Users → Invite User
   - Paste service account email
   - Select **"Developer"** access level (required for API operations)

### Credential Formats

The provider accepts credentials in two formats:

1. **File path**: Path to service account JSON key file
2. **JSON content**: Raw JSON string of the service account key

```hcl
provider "googlewallet" {
  credentials = "/path/to/service-account.json"  # File path
  # OR
  credentials = var.service_account_json         # JSON content
}
```

## Go Client Library

**Package:** `google.golang.org/api/walletobjects/v1`

### Service Initialization

```go
import "google.golang.org/api/walletobjects/v1"
import "google.golang.org/api/option"

// With credentials file
svc, err := walletobjects.NewService(ctx,
    option.WithCredentialsFile("path/to/key.json"))

// With credentials JSON
svc, err := walletobjects.NewService(ctx,
    option.WithCredentialsJSON([]byte(jsonContent)))
```

### Issuer Operations

```go
// Create
issuer, err := svc.Issuer.Insert(&walletobjects.Issuer{...}).Do()

// Get
issuer, err := svc.Issuer.Get(issuerId).Do()

// List
list, err := svc.Issuer.List().Do()

// Patch (partial update)
issuer, err := svc.Issuer.Patch(issuerId, &walletobjects.Issuer{...}).Do()

// Update (full replace)
issuer, err := svc.Issuer.Update(issuerId, &walletobjects.Issuer{...}).Do()
```

### Permissions Operations

```go
// Get
perms, err := svc.Permissions.Get(issuerId).Do()

// Update
perms, err := svc.Permissions.Update(issuerId, &walletobjects.Permissions{...}).Do()
```

## Rate Limits

- **20 requests per second** per issuer account
- Implement exponential backoff for retries
- Use caching where appropriate

---

## Important Constraints

1. **No Issuer Deletion**: Plan issuer naming carefully - they're permanent
2. **Permissions Overwrite**: Always fetch existing permissions before updating
3. **Service Account Needs Developer Role**: Admin role is insufficient for API access
4. **Callback URLs Must Be Public HTTPS**: Private/internal URLs not allowed
5. **SmartTapMerchantData Restricted**: Requires Google partnership program
6. **Demo Mode**: New accounts start in demo mode with limited publishing
